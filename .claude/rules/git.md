# Git ルール

## 安全チェックを迂回しない

以下のオプションは使用しないでください。pre-commit hook などの安全チェックを常に通してください。

| 禁止オプション | 理由 |
|---|---|
| `--no-verify` | pre-commit / commit-msg hook をスキップしてしまう |
| `--force` | リモート履歴を上書きしてしまう |

## GitHub リポジトリへのアクセス

`https://github.com/*` 配下のリポジトリにアクセスするときは `gh` コマンドを使用してください。

### 禁止されるアクセス方法

| 方法 | 許可 |
|---|---|
| `gh` コマンド | **許可** |
| WebFetch（`fetch_webpage` など） | 禁止 |
| MCP サーバー経由の直接アクセス | 禁止 |

## GH コマンド互換性ルール

- 非推奨 GitHub API に依存する `gh` サブコマンドは使用しない
- 非推奨警告（例：Projects classic 関連 GraphQL 警告）が出たら、そのコマンドの利用を停止する
- 編集操作（PR/issue 本文更新）では `gh api` の REST 呼び出しを優先する
- 更新後は必ず終了コードとレスポンスのリソース項目を確認する

## PR ルール

<!-- TODO: プロジェクトの PR 運用に合わせて編集してください -->

- PR タイトル：`[TicketID] <type>: <Summary>`（チケットなしの場合：`<type>: <Summary>`）
- PR タイトルは **英語のみ** で記述する
- ブランチ名は **英語のみ** で記述する
- マージ方式は squash merge を前提とする
- PR 作成時は `.github/pull_request_template.md` を使用する

### type 一覧

| type | 目的 |
|---|---|
| `feat` | 新機能 |
| `fix` | バグ修正 |
| `docs` | ドキュメントのみの変更 |
| `style` | コードの意味に影響しない変更（空白、整形など） |
| `refactor` | バグ修正でも新機能追加でもないコード変更 |
| `perf` | パフォーマンス改善 |
| `test` | テストの追加または変更 |
| `chore` | ビルドプロセスや補助ツールの変更 |
| `ci` | CI 設定の変更 |
