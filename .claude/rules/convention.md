# コーディング規約

## 言語ルール

| ドキュメント種別 | 言語 |
|---|---|
| ドキュメント（README など） | 日本語 |

| PR 本文、レビュー、レビュー返信 | 日本語 |
| 仕様書、チケット説明/コメント | 日本語 |

| PR タイトル、ブランチ名 | 英語 |
| ログメッセージ | 英語 |

## 共通ルール

- 既存のコーディングスタイルに従い、変更は最小限にする
- API キーなどの機微情報は環境変数またはその他の管理機構を使用する
- ハードコードされた値をコード内に直接書かない
- **コードを追加/変更する場合、対応するユニットテストは必須**
  - TDD を使用して、テストファーストで開発すること
  - 難しい場合は理由を明記すること
