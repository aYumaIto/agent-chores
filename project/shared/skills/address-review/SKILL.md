# レビュー指摘対応

## 利用する場面
- PR にレビューコメントがついたとき

## 必須ワークフロー

1. 変更方針をユーザーに確認する
2. 承認後にコード修正する
3. 返信ドラフトを作成しユーザー承認を得る
4. 承認後に投稿する

**方針未確認で修正開始禁止。承認なしで投稿禁止。**

## 手順

1. 未解決レビューコメントを取得する
2. 各指摘に対する変更方針をユーザーに提示し承認を得る
3. 承認された方針に従いコードを修正・コミットする
4. 返信を作成する：
   <!-- TODO: プロジェクトの返信フォーマットに合わせて編集 -->
   ```
   <返信本文>

   [<short-hash>](https://github.com/{owner}/{repo}/commit/<full-hash>)
   ```
   - コード変更なし（質問回答のみ）の場合はコミットリンク省略
5. 返信ドラフトをユーザーに提示し承認を得る
6. `gh api` で返信を投稿する
7. 自動レビュアー（bot）のスレッドのみ resolve する

## 条件

- 人間のレビュアーのスレッドは絶対に resolve しない（返信のみ）
- コメントと同じ言語で返信する
