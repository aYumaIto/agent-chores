# セキュリティ

## 機微情報

- APIキー・トークン・パスワードをソースコードにハードコードしない
- 機微情報は環境変数または専用管理機構を使用する
- ログ出力に機微情報を含めない

## コミット前チェック

- ハードコードされたシークレットが含まれていない
- 絶対パス・内部ホスト名が公開リポジトリに漏れていない
- ユーザー入力にバリデーションがある
- スタックトレースをユーザーに露出しない
- 依存関係に既知の脆弱性がない

## デバッグコード

- `console.log` / `debugger` / `binding.pry` / `print()` 等がコミットに含まれていない
- 意図的に残す場合はログレベル指定（`logger.debug` 等）を使用する
