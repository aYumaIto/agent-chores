# GitHub Copilot コードレビュー指示

GitHub Copilot がコードレビューを行う際の指示です。

## 重要: 必須ルール

**すべてのレビューコメントは次の形式に従うこと:**

```
[prefix]

<!-- TODO: プロジェクトに合わせて編集してください -->
<comment>
```

**ルール:**

<!-- TODO: プロジェクトのレビュー言語に合わせて編集してください -->

1. すべてのコメントを **日本語** で書く
2. すべてのコメントは **必ず** 下記プレフィックスのいずれかで開始する
3. プレフィックスなしのコメントは **厳禁**

## レビューコメントのプレフィックス

### 使用可能なプレフィックス（5種類のみ）

| prefix | 目的 | 使用する場面 |
|--------|------|-------------|
| **[must]** | 必須修正 | セキュリティ問題、バグ、重大な問題 |
| **[imo]** | 意見・提案 | より良い実装への提案（任意対応） |
| **[nits]** | 軽微な指摘 | コードスタイル、命名、タイポ |
| **[ask]** | 質問 | 実装意図の確認、不明点の確認 |
| **[fyi]** | 参考情報 | 関連ドキュメント、ベストプラクティス |

### 例

**正しい例:**

```
[must]

ユーザー入力を適切にサニタイズしてください。
```
```

**誤った例（禁止）:**

- プレフィックスなしでコメントを書く
- 指定言語以外でコメントを書く

## プレフィックス選択基準

レビューコメントを書くときは、次の基準でプレフィックスを選び、必ず付与すること。

| prefix | 選択基準 |
|--------|---------|
| **[must]** | セキュリティ脆弱性、バグ・不具合、データ損失リスク、重大なアーキテクチャ問題 |
| **[imo]** | より良い実装案、性能改善提案、可読性改善、将来拡張性を考慮した提案 |
| **[nits]** | 空行や空白の問題、変数/メソッド名の改善、コメント追加/修正、インデントや整形 |
| **[ask]** | 実装意図が不明、設計判断の理由を確認したい、要件確認が必要 |
| **[fyi]** | 関連ドキュメントの案内、新機能/API の情報提供、ベストプラクティスの共有 |
