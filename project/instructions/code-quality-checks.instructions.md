# コード品質チェック

## 編集後の確認事項

### デバッグコード

- デバッグ用コード（`console.log`, `debugger`, `print()`, `breakpoint()` 等）が残っていない

### フォーマット・リント

- プロジェクトにフォーマッタがあれば編集ファイルに実行する
- リンターがあれば実行し、新たなエラーを出さない

### テスト

- 対応テストが存在する場合、パスすることを確認する
- 公開APIシグネチャ変更時は呼び出し元とテストも更新する
