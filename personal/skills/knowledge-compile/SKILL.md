# ナレッジコンパイル

## 利用する場面

- 散在するドキュメントを構造化ファイルに変換したいとき

## 手順

1. ユーザーが指定したソースを読み、トピックを特定する
2. トピックごとにナレッジディレクトリへ `<kebab-case>.md` を作成する
3. 既存ナレッジと重複する場合は既存を更新する
4. 大量に生成する場合はユーザーに確認してから書き込む

ファイルフォーマット：

```markdown
# <トピックタイトル>

## いつ・なぜ使うか
<1〜3 行>

## 文脈
- 対象: <システム名>
- 時点: <YYYY-MM-DD>
- ソース: <元ドキュメントのパスやURL>

## 内容
<本文>
```

## 条件

- 1 ファイル 1 トピック
- 各ファイルは単独で意味が通じること
- 固有名詞・バージョン情報は省略しない
- 不確実な情報には `[要確認]` を付ける
- ソースドキュメント自体は変更しない
