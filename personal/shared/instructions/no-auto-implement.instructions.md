# 実装開始の制御

## ルール

- ユーザーが「実装して」「作って」「コードを書いて」など **明示的に実装を指示する** まで、コード変更・ファイル作成・コミットを行わないこと
- 実装指示がない段階では、以下に留める:
  - 調査・分析・設計の提示
  - Issue / チケットの作成・管理
  - 方針の確認・質問
  - ブランチ作成やファイル変更は **しない**

## 背景

事前に Issue 管理やチケット整理など、実装前の準備工程を挟む運用のため。
勝手に実装を始めると、準備が飛ばされてしまう。
