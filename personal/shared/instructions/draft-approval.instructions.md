# 外部サービスへの文章作成・更新時のルール

PR・チケットへのコメント、Issue/PR の作成、説明文の更新など、**外部サービスに対して文章を Create/Update する操作を行う前に、必ず下書きを出力してユーザーの許可を得ること。**

## ルール

- 許可なく投稿・更新を実行しない
- 下書きを提示し、ユーザーが承認してから実行する
- ただし、プロジェクトレベルで別の方法（例: 即時投稿可、テンプレート使用など）が指定されている場合は、そちらの指示を優先する

## 下書き提示前のセルフチェック

下書きを提示する直前（**書き直し・簡潔化での再提示を含む**）に、以下を必ず確認する:

- プロジェクトの convention / ルール（例: 日英併記、PR template 準拠、log 言語、branch 命名）
- 「簡潔にして」「短く」等のフィードバックで書き直したときも上記を再チェックする
  - 簡潔さは convention ルールより優先しない

確認をスキップして提示しないこと。
